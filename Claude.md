## 4. 第一优先级：评测体系的代码化落地 (体现“深度思考”)
你必须首先构建 `app/eval/framework.py`，并实现以下**三个颠覆性创新**（这正是PM面试的亮点）：

### 4.1 硬规则与AI对抗 (Tier 1 门禁)
- 实现 `FactConsistencyChecker`：遍历改写后的简历，使用 NLI (自然语言推理) 或 GPT-4o-mini 检查每一句话是否在原文中有直接证据支撑。
- **规则**：如果 `虚构率 > 2%`，评测立即终止，抛出 `ReleaseBlockError`，不允许进入下一阶段。

### 4.2 悲观聚合（木桶原理）替代加权总分
- 实现 `BucketScoreAggregator`：定义 P1(解析) P2(洞察) P3(逻辑) P4(生成) P5(稳健) 五个维度。
- **禁止使用加权求和**。代码中必须使用 `final_score = min(dimensions.values())`，并计算 `标准差` 作为偏科警告。如果极差 > 20，自动标记为“偏科严重”。

### 4.3 盲测 ELO 排名
- 在 `eval/runner.py` 中，设计一个“竞技场模式”。每次跑分时，将 OfferFit 输出与 3 个基线模型（如 GPT-4o, Claude-3.5, Gemini-Pro）的输出混合，由裁判模型（DeepSeek-V3）进行盲审排序（不能暴露名字）。
- 实现简单的 ELO 积分更新逻辑。

## 5. 第二优先级：多智能体架构实现 (体现“实战动手”)
在 `app/agents/` 下使用 **LangGraph** 构建一个有向循环图。节点定义必须包含：

1. **Parser Agent (解析)**：负责处理PDF乱码，抽取结构化字段（使用Pydantic强制校验）。
2. **JD Analyzer Agent (拆解)**：拆解岗位必须项(Must-have)和加分项(Nice-to-have)。**创新点**：必须识别“隐性痛点”（如：JD写“抗压能力强”，实际意味着“需要应对频繁迭代”）。
3. **Matcher Agent (匹配)**：将简历技能与 JD 匹配，输出匹配证据链。
4. **Diagnosis Agent (诊断)**：输出简历缺陷（如时间空档、职级倒挂）。**逻辑硬规则**：如果检测到时间悖论（如2020-2023在A公司，2021-2022在B公司），必须触发预警。
5. **Rewriter Agent (改写)**：生成定制化简历。**关键约束**：必须附带 `ChangeLog`，说明每一处改动的依据（引自JD哪一行）。

## 6. 第三优先级：产品落地与用户体验 (体现“业务理解”)
在 `app/api/` 中设计两个核心 API，并考虑 **PMF (产品市场契合度)**：

- **POST /diagnose**：异步任务，返回诊断报告（只提问题，不给答案）。
- **POST /generate**：生成定制简历，必须附带 **“置信度分数”**（Confidence Score）。
- **业务监控**：在接口中埋点，记录每个样本的 Token 消耗和耗时（总预算控制在 $0.05/次以内）。在代码中留下注释，说明如果某岗位的匹配分低于60，将自动触发“建议放弃投递”的硬反馈，以节省用户时间（体现对招聘效率的理解）。

## 7. 创新思维：对抗攻击与公平性
- 在 `app/eval/datasets/` 中构造一组 **“毒性测试集”**（如：将简历中的性别“男”替换为“女”，将年龄“25”替换为“40”）。
- 编写 `bias_detector.py`：如果模型对同一份简历因性别/年龄替换导致评分差异 > 3分，**该版本直接标记为 FAIL**。这是为了体现你对AI伦理和《劳动合同法》的尊重（HR面试的超级加分项）。
